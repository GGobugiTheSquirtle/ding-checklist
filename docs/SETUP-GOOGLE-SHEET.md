# 구글시트 연동 가이드

체크리스트 마스터 데이터를 **구글시트로 관리**하려면 아래 절차를 따르세요.
설정 후에는 시트를 수정하면 앱에 자동 반영됩니다 (캐시 없음).

> 구글시트 연동은 **선택 사항**입니다. 설정하지 않으면 번들된 `data/master-data.json`을 사용합니다.

---

## 왜 구글시트인가?

- **실시간 갱신**: 원작자가 시트를 수정하면 모든 사용자의 앱이 다음 새로고침 시 자동 반영
- **편집 편의**: 스프레드시트 UI로 추가/수정/정렬
- **인증 불필요**: "웹에 게시" CSV 링크를 쓰므로 사용자는 별도 로그인 없음
- **읽기 전용**: 사용자는 마스터를 변경할 수 없음 (개인 진행도는 브라우저 localStorage에 저장)

---

## 1단계 — 시트 생성

1. [Google Sheets](https://sheets.google.com) 에서 새 스프레드시트 생성
2. 첫 행(헤더)에 **정확히** 다음 5개 컬럼을 입력:

| A | B | C | D | E |
|---|---|---|---|---|
| `category` | `section` | `id` | `text` | `completed` |

3. `data/master-sheet.csv` 파일을 열어 전체 내용을 복사 → 시트에 붙여넣기
   - 엑셀/Numbers가 자동으로 열리면 "가져오기" 메뉴에서 CSV로 입력
   - 구글시트에서 바로: `파일 > 가져오기 > 업로드` → `data/master-sheet.csv` 선택 → `데이터를 스프레드시트에 대체`

4. (선택) 시트 탭 이름을 `master` 로 변경. 여러 시트를 쓸 거면 필수.

---

## 2단계 — 웹에 게시 (CSV)

1. 구글시트 상단 메뉴: `파일 > 공유 > 웹에 게시`
2. 다음 두 가지를 설정:
   - **링크 탭** (Link)
   - 형식: **쉼표로 구분된 값 (.csv)** (Comma-separated values)
   - 게시할 시트: 방금 만든 `master` 시트
3. `게시` 버튼 클릭 → URL이 생성됨

생성되는 URL 형태:

```
https://docs.google.com/spreadsheets/d/e/2PACX-1vR.../pub?gid=0&single=true&output=csv
```

> ⚠️ `/edit` 으로 끝나는 일반 공유 링크는 사용할 수 없습니다. 반드시 **"웹에 게시"** 후 나오는 `/pub?...&output=csv` URL이어야 합니다.

---

## 3단계 — 앱에 URL 주입

`index.html` 파일을 열어 `<script src="app.js"></script>` 태그 **바로 앞**에 다음 스크립트를 추가:

```html
<script>
  window.__CONFIG__ = {
    sheetCsvUrl: "여기에_2단계에서_복사한_CSV_URL_붙여넣기"
  };
</script>
<script src="app.js"></script>
```

전체 예시:

```html
<script>
  window.__CONFIG__ = {
    sheetCsvUrl: "https://docs.google.com/spreadsheets/d/e/2PACX-1vR.../pub?gid=0&single=true&output=csv"
  };
</script>
<script src="app.js"></script>
```

저장 후 페이지 새로고침 → 시트의 내용이 로드되면 성공.

시트 로드 실패 시 자동으로 `data/master-data.json` 번들본으로 폴백하므로 오프라인/오프라인에서도 앱은 동작합니다.

---

## 컬럼 규칙

| 컬럼 | 필수 | 설명 | 예시 |
|------|------|------|------|
| `category` | ✅ | 최상위 카테고리명. 동일 이름을 반복 사용하면 같은 카테고리로 묶임 | `메인`, `공통`, `외전` |
| `section` | ❌ | 섹션명(선택). 비워두면 카테고리 직속 아이템 | `제1부 <Another Eden — 시공을 넘는 고양이>` |
| `id` | ✅ | 고유 ID. 영문/숫자/하이픈. **한 번 정해지면 바꾸지 말 것** (사용자 진행도가 id로 연결됨) | `c1-s0-i0` |
| `text` | ✅ | 항목 본문 | `메인 퀘스트 제1~26장` |
| `completed` | ❌ | 무시됨 (앱은 항상 `FALSE`로 초기 로드. 개인 진행도는 브라우저 저장) | `FALSE` |

### ID 네이밍 컨벤션

현재 번들본 네이밍: `c{카테고리순서}-s{섹션순서}-i{아이템순서}` 또는 카테고리 직속일 때 `c{순서}-i{순서}`.

**새 항목 추가 시 기존 ID 체계를 따라 충돌하지 않게 작성**. 예:
- `메인 > 제3부 최종편`의 마지막 아이템이 `c1-s12-i50`이라면 새 아이템은 `c1-s12-i51`
- 완전 새 카테고리라면 `c20-...` 처럼 순서를 이어 붙임

### 기존 항목 수정 시 주의

- `text` 변경은 OK — 진행도는 id로 연결되므로 표기만 바뀜
- `id` 변경은 절대 금지 — 사용자 진행도가 전부 깨짐
- 항목 삭제도 주의 — 삭제된 id의 진행도는 보관되지만 UI에 노출 안 됨

## 문제 해결

### "구글시트 로드 실패" 토스트가 뜸
- URL이 `/pub?...&output=csv` 로 끝나는지 확인
- 시트가 "웹에 게시" 상태인지 확인 (비공개면 실패)
- 브라우저 DevTools 콘솔(F12)에서 구체적 에러 메시지 확인

### 새로고침해도 갱신이 안 됨
- 구글시트의 변경사항이 CSV 퍼블리시에 반영되는 데 최대 5분 지연
- 시도: 시크릿 창에서 열거나 URL 뒤에 `&t=<타임스탬프>` 파라미터를 추가

### CSV 파싱 에러
- 헤더 첫 행이 `category, section, id, text, completed` 순서인지 확인
- 텍스트 안에 따옴표/쉼표/개행이 있어도 구글시트가 자동으로 이스케이프하므로 OK
- 빈 행은 자동으로 무시됨

## 고급: Apps Script 로 마스터 업데이트 자동화 (선택)

매번 수동으로 시트를 수정하기 귀찮다면 [Apps Script](https://script.google.com) 로 자동화 가능합니다.
예: 네이버 카페에서 최신 체크리스트를 긁어와 시트에 밀어넣는 스크립트. 관심 있으면 별도 문의.
